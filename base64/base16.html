ğŸ” Base16 (Hexadecimal) Working Principle

  • Character Set: Base16 uses 16 symbols 0-9 and A-F to represent bytes
  • Grouping: Each byte (8 bits) is split into two 4-bit nibbles, each nibble maps to one hexadecimal digit
  • Encoding Example: Byte 0x4F â†’ Binary 0100 1111 â†’ Hexadecimal "4F"
  • Decoding: Decoding groups every two hexadecimal digits into one byte, spaces and newlines are ignored
  • Security Note: Hexadecimal is a representation method, not encryption, it does not provide confidentiality

Notes

  • Case Sensitivity: Hexadecimal letters are case-insensitive, A-F and a-f are equivalent
  • Length Requirement: Hexadecimal string length must be even, odd length will cause error
  • Character Limit: Can only contain 0-9, A-F (or a-f) characters, other characters will be ignored
  • Whitespace Handling: Spaces, newlines and other whitespace characters in input will be automatically ignored
  • Encoding Efficiency: Hexadecimal encoding will double the data size
